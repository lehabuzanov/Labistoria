# Параллельный корпус списков

Streamlit-приложение для демонстрации параллельного корпуса нескольких списков одного текста на материале XML-TEI. Проект поддерживает:

- загрузку и постоянное хранение XML-TEI файлов;
- автоматическое выравнивание списков по одному слову;
- показ и фильтрацию типов разночтений;
- ручную правку ошибок выравнивания;
- сохранение сессии между перезапусками;
- экспорт результата обратно в XML-TEI в виде отдельного TEI-документа параллельного корпуса.

Репозиторий: <https://github.com/lehabuzanov/Labistoria>

## Хранилище данных

После первого импорта приложение создаёт каталог `storage/`:

- `storage/uploads` — исходные XML-файлы;
- `storage/parsed` — разобранные представления документов с токенами;
- `storage/corpus.db` — SQLite-база с метаданными, сохранёнными выравниваниями и ручными правками;
- `storage/exports` — экспортированные TEI-документы параллельного корпуса.

За счёт этого после закрытия и повторного запуска Streamlit повторная загрузка файлов не нужна.

## Запуск локально

1. Создайте виртуальное окружение.
2. Установите зависимости:

```bash
pip install -r requirements.txt
```

3. Запустите приложение:

```bash
python -m streamlit run streamlit_app.py
```

4. Откройте адрес, который покажет Streamlit в терминале.

## Самопроверка

Перед публикацией можно прогнать сценарий самопроверки:

```bash
python scripts/self_check.py
```

Скрипт:

- импортирует пять исходных XML;
- строит автоматическое выравнивание;
- вносит ручные правки;
- сохраняет и перечитывает сессию;
- экспортирует результат в XML-TEI;
- проверяет, что данные и экспорт реально существуют;
- проверяет наличие локальных шрифтов, их встраивание в CSS и покрытие ключевых символов рукописного и интерфейсного слоёв.

## Шрифты и визуальный стиль

Для рукописного слоя используется `Monomakh`, а редкие символы и пунктуация страхуются fallback `Noto Serif`. Шрифты подключаются в CSS отдельными URL из репозитория, поэтому интерфейс остаётся лёгким для Streamlit Cloud и не зависит от системных подстановок браузера. Для интерфейса используются `Old Standard` и `Oranienbaum`: они дают старостильный вид, но сохраняют современную читаемость русских букв в заголовках, кнопках и служебных текстах.

Интерфейс оформлен под академический манускрипт: добавлены орнаментальные узоры, пергаментные подложки, цветовая дифференциация типов разночтений и отдельная “рукописная” таблица корпуса вместо стандартного `st.dataframe`, чтобы Streamlit больше не ломал шрифтовой слой.

## Поток работы

1. Импортируйте XML из корня проекта или загрузите новые файлы.
2. Выберите главный список и диапазон листов.
3. Выберите остальные списки и соответствующие диапазоны.
4. Запустите автоматическое выравнивание.
5. Просматривайте и фильтруйте разночтения.
6. Исправляйте ошибки вручную через блок корректировки строк.
7. Продолжайте работу в следующий сеанс без новой загрузки файлов.
8. Экспортируйте результат в TEI.

## Замечание по исходным данным

В исходных XML встречается важная особенность: декларация кодировки внутри файла может говорить `UTF-16`, тогда как фактические байты читаются как UTF-8. Импортёр специально учитывает такие расхождения и не падает на них.
